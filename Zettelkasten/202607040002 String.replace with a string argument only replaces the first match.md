`String.prototype.replace(searchValue, newValue)` only replaces **every** occurrence when `searchValue` is a regex with the global flag (`/pattern/g`). Passed a plain string, it replaces only the **first** match and leaves every later occurrence untouched:

```js
'<a><b><c>'.replace('<', '&lt;')   // '&lt;a><b><c>'  — only the first '<' encoded
'<a><b><c>'.replace(/</g, '&lt;')  // '&lt;a&lt;b&lt;c'  — all of them
```

As a sanitizer, the string form of `.replace()` is a common real-world bug: it looks like it does the job because the first tag in a simple test case gets encoded, but any payload preceded by a throwaway pair of angle brackets sails through untouched. The fix is trivial (`/</g`), which is exactly why it's worth specifically testing for on any app that appears to "encode angle brackets" — try two pairs, not one.
